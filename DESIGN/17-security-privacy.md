# 보안 및 개인정보

## 위협 모델
- 주요 리스크: 계정 도용, 거래 사기, 데이터 유출.

## 인증
- 인증 흐름: OAuth 기반 + 2FA 선택.
- 세션 규칙: 토큰 갱신, 동시 접속 제한.

## 데이터 보호
- 암호화: 전송 TLS, 저장 AES.
- 접근 제어: 최소 권한 원칙.

## 컴플라이언스
- 지역 규정: GDPR/지역 개인정보법 준수.
- 연령 제한: 청소년 보호와 채팅 제한.

## 세부 설계 (BitCraft 참고)

### 인증/세션
- Identity 기반 인증, 24시간 내 재인증 필요.
- sign_in 시 중복 세션/차단/큐 상태 검증.

### 역할/권한
- Role 계층(Player -> Mod -> GM -> Admin 등)으로 작업 제한.
- 공유 테이블로 전 모듈 권한 동기화.

### 서버 권위
- ServerIdentity 검증으로 에이전트/관리 리듀서 실행 보호.
- dev 모드 권한 우회는 운영에서 금지.

### 악용 차단
- BlockedIdentity, UserModerationState로 영구/기간 차단.
- 권한/클레임/건물 접근은 PermissionState로 계층 검증.
