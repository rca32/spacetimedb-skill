# 기술 스택 및 빌드

## 클라이언트
- 엔진: 언리얼 또는 유니티.
- 플랫폼: PC 우선, 추후 콘솔.
- 클라이언트 SDK: TypeScript 우선(웹/툴링 및 엔진 브리지).

## 서버
- 런타임: Rust + SpacetimeDB 모듈.
- 데이터베이스: 테이블 기반 상태 저장 + 캐시.

## CI/CD
- 빌드 파이프라인: PR 빌드, 야간 통합, 자동 배포.
- 배포 대상: 리전별 클러스터.

## 모듈 빌드
- 게임/글로벌 모듈을 각각 WebAssembly로 빌드.
- 리듀서/테이블 스키마 자동 검증.

## 툴링
- 에디터 도구: 레벨/퀘스트/경제 관리 도구.
- 분석: KPI 대시보드와 로그 분석.

## 세부 설계 (BitCraft 참고)

### 런타임/언어
- Rust 2021 + SpacetimeDB 런타임, spacetimedb-bindings 사용.
- 모듈은 cdylib로 컴파일, WASM 배포.

### 빌드 최적화
- 릴리스 빌드는 opt-level=s 및 LTO 적용.
- 테이블 구조는 정렬/패딩 최소화로 직렬화 비용 절감.

### 환경 구성
- dev/qa/staging/prod 환경별 config 분리.
- dev 모드에서 인증/권한 우회(운영 금지).
