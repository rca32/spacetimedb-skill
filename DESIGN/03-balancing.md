# 밸런싱

## 원칙
- 협동 중심 체험을 해치지 않는 수평적 선택지 제공.
- 과도한 메타 고착 방지, 다양한 빌드가 동등한 효율.
- 성장 속도는 완만하고, 플레이 시간 차이를 완충.
- “재미의 곡선” 유지: 학습(발견) → 숙련(성취) → 확장(사회/경제) 단계가 분명.
- 플레이어 손실 회피: 실패/손실이 학습으로 전환되도록 보상/복구 경로 제공.

## 코지 정량 기준
- 전투 패배 손실: 인벤토리 드롭 없음, 장비 내구도 손실 5% 이하.
- 회복 시간: 사망 후 3분 내 전투 복귀 가능(이동+회복 포함).
- 실패 비용: 동일 활동 1회 재도전 비용이 해당 세션 평균 획득량의 20% 이내.
- PvP 손실 상한: 비아레나 PvP는 순위/칭호 중심, 자원/아이템 손실은 제한적.

## 성장 곡선
- 레벨 XP 곡선: 초반 빠른 성취, 중반 완만, 후반은 계단형.
- 장비 스케일링: 재료 등급과 제작 숙련도의 기여도 분리.
- 세션 목표 설계: 30~60분 내 “완결감 있는 작업 1~2개” 달성 가능.
- 스킬 성장: 사용 기반 + 보너스 목표(멘토링/협동)로 사회적 학습 유도.

## 솔로 성장 범위
- 솔로 플레이는 메인 루프(탐험/제작/거래/기본 전투)의 70% 수준까지 도달 가능.
- 최상위 티어(대규모 공성/레이드/대형 클레임 확장)는 파티/길드 요구.

## 전투 밸런스
- 목표 TTK: 솔로 기준 일반 몬스터 6-10초, 엘리트 20-40초.
- 파티 스케일링: EnemyScalingState로 체력 위주 증가, 공격력은 완만 상승.
- 역할군 균형: 탱/딜/서포트가 30/50/20 기여.
- 읽기 쉬운 전투: 상태 이상/약점/치명타가 시각적으로 명확해야 함.
- “위기-회복” 리듬: 회복 타이밍을 제공해 스트레스-해소 사이클 형성.

## 경제 밸런스
- 인플레이션 제어: 거래세, 수리비, 제작 소모품을 핵심 싱크로 사용.
- 싱크/소스 비율: 주간 기준 싱크 55%, 소스 45% 목표.
- 희소성 곡선: 초반 핵심 자원은 안정적, 중/후반 고급 자원은 변동성 제공.
- 생활비 압박 완화: 클레임 유지비는 “활동적 플레이 시 감당 가능” 수준 유지.

## 건축/클레임 감가
- 무클레임 건축물: 하루 20~30% 내구도 감소, 3~5일 내 자연 소멸 목표.
- 감가 수치는 balance_params로 조정하며 라이브 튜닝 대상.

## PvP 밸런스
- 매칭 제약: 레벨/장비 점수 범위 매칭, 파티 규모 보정.
- 장비 정규화: 아레나는 부분 정규화, 필드 PvP는 완화.
- 패배 비용 최소화: 대량 손실 대신 명예/랭크 기반 보상 구조 유지.

## 파라미터 기본값(초안)
- 치명타 확률 상한: 35%.
- 회피 상한: 25%.
- 속성 저항 상한: 60%.
- 군중제어(CC) 연속 저항: 동일 CC 반복 시 효과 감소(디미니싱 리턴).
- 유틸 스킬 상한: 팀당 핵심 유틸(버프/디버프) 적용 수 제한.

## 라이브 튜닝
- 핫픽스 파라미터: 드랍률, 수리비, 스킬 계수.
- 가드레일: 하루 변경폭 10% 이내, 주간 누적 25% 이내.
- 롤백 정책: 지표 악화 5% 초과 시 즉시 이전값 복귀.
- 세션 기준 튜닝: 동일 파라미터가 한 세션 내 급격히 변하지 않도록 고정.
- 파라미터 버전: balance_epoch 키를 증가시키고, 변경 시 클라이언트 강제 리로드.

## 세부 설계 (BitCraft 참고)

### 재생/피로 밸런스
- 재생 구조: Active + Passive 이중 재생, 패시브는 마지막 감소 후 쿨다운 경과 + 비전투 상태에서만 활성.
- 굶주림: Satiation 0 시 Starving 디버프 부여, 음식으로 해제.

### 건축/클레임 유지
- 유지비 = (타일 유지비 + 건축 유지비 합)로 계산, 공급 부족 시 건축물 체력 감가.
- 무클레임 건축은 빠른 감가로 월드 정리 유도.

### 전투/위협
- 위협 점수 기반 타겟팅, 도발은 최고 위협으로 equalize 후 가산.
- 다중 플레이어 참여 시 적 스케일링으로 난이도 보정.

### 경제/거래
- 직접 거래: 세션 잠금/타임아웃으로 리스크 최소화.
- 주문장: 가격 우선 매칭 + 시간 우선(tiebreaker), escrow 코인으로 과열 완충.
- 바터 스톨: 고정 교환 + 재고 제한으로 희소성 유지.

## 재미 포인트 설계

### 발견의 재미
- 지도/바이옴/자원 다양성으로 “다음에 갈 곳”이 항상 보이게.
- NPC 소문/퀘스트가 탐험 방향을 제시하도록 설계.

### 성취의 재미
- 프로젝트 사이트 완료, 클레임 확장, 거래 성사 같은 명확한 완료감 제공.
- 성장 마일스톤(제작 등급/기술 해금)마다 체감 변화를 강조.

### 사회적 재미
- 협동 건축/거래/방어 이벤트로 공동 목표 제공.
- 길드/엠파이어 내 역할 차별화(수집/제작/전투/외교).

### 경제적 재미
- 시장에서 “가격 읽기”와 “공급 기회”가 존재하도록 변동성 유지.
- NPC 상점/바터 스톨로 기본 공급 안정화.

## 적응형 밸런싱 (AI/ML 적용 여지)

### 적용 가능한 영역
- 드랍률/재생성 주기/거래 수수료/퀘스트 보상(소폭).
- NPC 주문 구성(항상 제공 + 선택형 비율).
- 지역 이벤트 빈도(참여율 기반).

### 입력 신호
- 리텐션/세션 길이/완료율/실패율/경제 지표.
- 거래 스프레드/시장 유동성/코인 집중도.
- 전투 TTK/사망률/회복 소요 시간.

### 안전 가드레일
- 변경 폭 제한(일 10%, 주 25%).
- 지역/세션 단위로 변경 사항 고정(세션 중 급변 금지).
- 고위험 파라미터(전투 계수/핵심 경제)는 수동 승인 필요.
- 변경 사유/근거 로그 저장 + 롤백 버튼 제공.

### 운영 모드
- 실험 모드: 소규모 리전/테스트 샤드에서만 적용.
- 단계 확장: 5% → 25% → 100% 순차 적용.
